# EBG — NoLiMa-based RAG Evaluation

[← Repo overview](../README.md)

RAG Top-K evaluation pipeline built on the NoLiMa benchmark. Embeds haystack chunks into Qdrant, retrieves the top-K most similar chunks per question, feeds them to multiple LLMs as context, and evaluates whether the model identifies the correct needle.

> **Note on result directories:** `rag_experiment_results/` is the output directory of `run_rag_experiment.py`. Both pre-existing results and fresh runs write to the same location. Plot scripts read from `rag_experiment_results/tables/rag_results_raw.csv`.

> **Note on pre-existing scored hit JSONs:** `results/NeedleRanking/<context_length>/` contains pre-existing per-question Qdrant scored hit JSONs. There are **19 context-length directories** (10000 to 1000000), each with **8 JSON files** — one per `<task_id>_<hop>` combination where task_id ∈ {0402, 0402Inv, 0405, 0405Inv} and hop ∈ {onehop, twohop}. Generated by `run_retrieval_ranking.py --save_hits`. Fresh runs default to the same directory so new hits coexist with pre-existing ones.

> **Model directory naming:** `.` is replaced with `-` in directory names. `gpt-4.1` → `results_gpt-4-1`, `gpt-5-2025-08-07` → `results_gpt-5-2025-08-07`, etc.

## Setup

### 1. Set PYTHONPATH

Scripts that import `ragwithtopk` require the package on the Python path:

```bash
export PYTHONPATH="$PWD/NoLiMa_based_RAG:$PYTHONPATH"
```

### 2. Set API keys

```bash
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 3. Install dependencies

```bash
python3 -m pip install -r NoLiMa_based_RAG/requirements.txt
```

### 4. Start Qdrant (required for Steps 1–2 only)

```bash
docker run -p 6333:6333 qdrant/qdrant
```

## Pipeline

Run all commands from **`EBG_repo/`**.

---

### Step 1 — Build Questions (`build_questions.py`)

Extracts questions from the NoLiMa needle set and adds OpenAI embeddings.

```bash
python3 NoLiMa_based_RAG/scripts/build_questions.py \
    --needle_set  datasets/NoLiMa/needlesets/needle_set.json \
    --out         NoLiMa_based_RAG/questions/questions.json \
    --reasoning_type commonsense_knowledge \
    --qtype       all \
    --model       text-embedding-3-small
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--needle_set` | *(required)* | Path to `needle_set.json` |
| `--out` | *(required)* | Output JSON with questions + embeddings |
| `--reasoning_type` | `commonsense_knowledge` | Task reasoning type |
| `--qtype` | `all` | `all`, `onehop`, or `twohop` |
| `--model` | `text-embedding-3-small` | OpenAI embedding model |
| `--batch_size` | `32` | Embedding batch size |
| `--overwrite_embeddings` | off | Re-compute embeddings even if already present |

**Output:** `NoLiMa_based_RAG/questions/questions.json` (8 questions with fields: `task_id`, `question_key`, `test_key`, `question`, `relevant_needle_id`, `q_id`, `embedding`)

---

### Step 2 — Run Retrieval Ranking (`run_retrieval_ranking.py`)

Queries Qdrant for every (source_file, question) pair and saves scored hit lists. Requires Qdrant running and the vector store already populated.

```bash
python3 NoLiMa_based_RAG/scripts/run_retrieval_ranking.py \
    --config         NoLiMa_based_RAG/configs/ck.yaml \
    --questions_json NoLiMa_based_RAG/questions/questions.json \
    --out_dir        NoLiMa_based_RAG/results/NeedleRanking \
    --save_hits
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--config` | *(required)* | YAML config (Qdrant URL, collection, embedding model, text_files) |
| `--questions_json` | *(required)* | Questions JSON with embeddings |
| `--out_dir` | `NoLiMa_based_RAG/results/NeedleRanking` | Output directory |
| `--source_files` | *(derived from config `text_files`)* | Override list of source file names |
| `--limit_qids` | all | Run only first N questions |
| `--save_hits` | off | Write per-question scored hit JSON files (needed for Step 3) |
| `--continue_on_error` | off | Skip errors instead of failing |
| `--extra_needles` | `false` | Include multi-needle tasks |

**Outputs in `<out_dir>/` (default: `NoLiMa_based_RAG/results/NeedleRanking/`):**
- `needle_rank_summary.json` — needle rank per (source_file, q_id); written as `NeedleRanking/needle_rank_summary.json`
- `<context_length>/<source_file>_<q_id>.json` — per-question scored hit JSON files (written if `--save_hits`)
- `errors.json` — any errors encountered

**`configs/ck.yaml` key settings:**

```yaml
qdrant_url:        "http://localhost:6333"
qdrant_collection: "book3"
embedding_model:   "text-embedding-3-small"
text_files:        # haystack files under datasets/NoLiMa/haystacks/
  - "datasets/NoLiMa/haystacks/rand_shuffle_10000/rand_book_1.txt"
  - ...
```

---

### Step 2b — Generate Full Binary Accuracy Table (`compute_rag_accuracy_from_full_summary.py`)

Joins the LC baseline accuracy (`results_summary_full.csv`) with needle retrieval ranks (`needle_rank_summary.json`) to produce a per-topk binary-gated accuracy table used by `needlewise_best_topk_pdf.py`.

> **Note:** The output CSV is gitignored — colleagues must run this step locally to regenerate it.

```bash
python3 NoLiMa_based_RAG/scripts/compute_rag_accuracy_from_full_summary.py \
    --base-csv          NoLiMa_based_RAG/results/results_summary_full.csv \
    --needle-json       NoLiMa_based_RAG/results/NeedleRanking/needle_rank_summary.json \
    --out-csv           NoLiMa_based_RAG/tables/rag_accuracy_full_binary_from_results_summary_full.csv \
    --missing-report-csv NoLiMa_based_RAG/results/missing_rows_results_summary_full_vs_needle.csv
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--base-csv` | `NoLiMa_based_RAG/results/results_summary_full.csv` | LC accuracy CSV (`model`, `context_length`, `reasoning_hop`, `needle_id`, `character`, `depth`, `combined_accuracy`, `answer_accuracy`, `evidence_accuracy`) |
| `--needle-json` | `NoLiMa_based_RAG/results/NeedleRanking/needle_rank_summary.json` | Needle rank JSON from Step 2 (`source_file`, `q_id`, `task_id`, `question_key`, `needle_rank_1based`, `num_scored`, `total_vector_count`) |
| `--out-csv` | `NoLiMa_based_RAG/tables/rag_accuracy_full_binary_from_results_summary_full.csv` | Output binary accuracy table |
| `--missing-report-csv` | `NoLiMa_based_RAG/results/missing_rows_results_summary_full_vs_needle.csv` | Rows with no matching needle rank (diagnostic) |

**How it works:** For each row in the base CSV (one per model/context/needle/depth), it looks up the needle's rank in the retrieval results. For every `topk` from 1 to `total_vector_count`, it sets `retrieval_hit_at_topk = 1` if `needle_rank ≤ topk`, else 0. This creates a row per `(model, context, needle, topk)` combination.

**Output columns:** `model`, `context_length`, `reasoning_hop`, `needle_id`, `character`, `depth`, `topk`, `needle_rank_1based`, `total_vector_count`, `retrieval_hit_at_topk`, `lc_combined_accuracy`, `rag_combined_accuracy`, `q_id`, `source_file`

**Output:** `NoLiMa_based_RAG/tables/rag_accuracy_full_binary_from_results_summary_full.csv`

### Step 3 — Run RAG Experiment (`run_rag_experiment.py`)

Loads scroll-all hits, selects top-K chunks per budget, calls LLMs, evaluates responses. **Does not import `ragwithtopk`** — self-contained.

**Pre-requisites:**
- `NoLiMa_based_RAG/results/NeedleRanking/` populated from Step 2 (`--save_hits`), or pre-existing data already present

```bash
# Dry run first to verify paths and pending tasks
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py --dry-run

# List available models
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py --list-models

# Run all models, all context lengths
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py

# Run one model
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py \
    --model claude-sonnet-4-5-20250929

# Run one context length
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py \
    --context-length 200000

# Run OpenAI and Anthropic providers concurrently
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py --parallel
```

| Argument | Description |
|----------|-------------|
| `--model` | Run one model only |
| `--context-length` | Run one context length only |
| `--dry-run` | Show pending tasks without calling APIs |
| `--list-models` | Print available models and exit |
| `--parallel` | Run OpenAI and Anthropic providers concurrently |

**Available models:**

| Model ID | Provider | Max context |
|----------|----------|-------------|
| `gpt-4o` | openai | 128K tokens |
| `gpt-4.1` | openai | 1M tokens |
| `o3-mini-2025-01-31` | openai | 200K tokens |
| `gpt-5-2025-08-07` | openai | 400K tokens |
| `claude-sonnet-4-20250514` | anthropic | 200K tokens |
| `claude-sonnet-4-5-20250929` | anthropic | 200K tokens |

**Default context lengths:** 200K, 400K, 500K, 600K, 1M characters

**Hardcoded paths** (all relative to `EBG_repo/`):

```
NoLiMa_based_RAG/results/NeedleRanking/               ← scored hit JSONs (Step 2 output / pre-existing)
NoLiMa_based_RAG/tables/topk_character_budget_twohop.csv
NoLiMa_based_RAG/questions/questions.json
datasets/NoLiMa/needlesets/needle_set.json
NoLiMa_based_RAG/rag_experiment_results/              ← results written here
```

**Output path pattern:**
```
NoLiMa_based_RAG/rag_experiment_results/
  results_<model-with-dots-as-dashes>/
    commonsense_knowledge/
      rand_shuffle_<context_length>/
        <q_id>/
          <model>_rand_book_1_<q_id>.json
```

### Step 3b — RAG Evaluation with Config (`run_rag_eval.py`)

Alternative evaluation script fully driven by `configs/rag_eval.yaml`. Supports multiple models in one run and auto-discovery of available context lengths.

```bash
# Standard run
python3 NoLiMa_based_RAG/scripts/run_rag_eval.py \
    --config NoLiMa_based_RAG/configs/rag_eval.yaml

# Dry run (fake LLM responses — useful for testing paths)
python3 NoLiMa_based_RAG/scripts/run_rag_eval.py \
    --config NoLiMa_based_RAG/configs/rag_eval.yaml \
    --dry-run
```

**Key `configs/rag_eval.yaml` fields:**

```yaml
scroll_all_dir:         "NoLiMa_based_RAG/results/NeedleRanking"
needle_set_path:        "datasets/NoLiMa/needlesets/needle_set.json"
questions_json:         "NoLiMa_based_RAG/questions/questions.json"
output_dir:             "NoLiMa_based_RAG/data/evaluation"
top_k_values:           [5, 10, 20, 50, 100]
auto_discover_contexts: true   # scans scroll_all_dir for numeric subdirectories
rand_books:             [1]
models:
  - name: "gpt-4.1"
    provider: "openai"
task_ids:               ["0402", "0402Inv", "0405", "0405Inv"]
test_key:               "T01_C02"
limit_context_lengths:  [10000]   # restrict to subset (comment out to run all)
limit_top_k_values:     [5, 10, 15, ..., 75]
```

**Output path:**
```
<output_dir>/results_<model>_w_rag/<context_length>/<eval_name>.json
```

### Step 4 — Retrieve All With Scores (`retrieval_with_scores.py`)

Diagnostic script that retrieves all scored hits from Qdrant for a single query and prints the full scored list to stdout. Calls `retrieve_all_with_scores` (no top-K limit) — useful for inspecting the complete ranking order.

```bash
# Using a pre-computed embedding from questions.json
python3 NoLiMa_based_RAG/scripts/retrieval_with_scores.py \
    --config         NoLiMa_based_RAG/configs/ck.yaml \
    --questions_json NoLiMa_based_RAG/questions/questions.json \
    --q_id           0402_onehop \
    --source_file    200000_rand_book_1

# Using a raw text query
python3 NoLiMa_based_RAG/scripts/retrieval_with_scores.py \
    --config  NoLiMa_based_RAG/configs/ck.yaml \
    --query   "Which character cannot eat Brandade?" \
    --topk    10
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--config` | *(required)* | YAML config (Qdrant URL, collection, embedding model) |
| `--questions_json` | — | Path to questions.json (needed with `--q_id`) |
| `--q_id` | — | Question ID to look up (requires `--questions_json`) |
| `--query` | — | Raw text query (alternative to `--q_id`) |
| `--topk` | `5` | Number of results (passed to Qdrant) |
| `--source_file` | — | Filter to one source file |
| `--needle_item_id` | — | Filter to one needle ID |

Requires Qdrant running and `OPENAI_API_KEY`. Requires PYTHONPATH (`ragwithtopk`).

Bar charts comparing cosine similarity of original vs. rephrased needle variants to the question. Adds `NoLiMa_based_RAG/` to `sys.path` automatically — no PYTHONPATH needed.

```bash
python3 NoLiMa_based_RAG/scripts/characters_pdf.py \
    --out-dir NoLiMa_based_RAG/plots \
    --dpi     300
```

| Argument | Default | Description |
|----------|---------|-------------|
| `--out-dir` | `NoLiMa_based_RAG/plots` | Output directory |
| `--dpi` | `300` | PDF DPI |

**Outputs:**
- `<out-dir>/characters_similarity_scores.pdf`
- `<out-dir>/characters_rephrased_similarity_comparison.pdf`

### `get_embeddings_of_needles.py`

Diagnostic script that prints needle–question cosine similarities to stdout and saves PNG plots.

> **Warning:** Line 46 contains a literal placeholder path `"RAG_Topk/data/plots/<plot_name>.png"` — the first plot will be saved to a file named literally `<plot_name>.png`. Use `characters_pdf.py` for clean output instead.

```bash
python3 NoLiMa_based_RAG/scripts/get_embeddings_of_needles.py
```

No arguments. Requires `OPENAI_API_KEY`.

**Outputs:** (script uses hardcoded plot paths — see warning above)
- `RAG_Topk/data/plots/<plot_name>.png` *(literal broken placeholder)*
- `RAG_Topk/data/plots/CharacterWOGender.png`

## `ragwithtopk` Package

Internal Python package. Must be on `PYTHONPATH` for scripts that import it.

```
ragwithtopk/
├── embeddings/
│   └── openai_embedder.py        ← OpenAI text-embedding-3-* wrapper
├── vectorstore/
│   └── qdrant_store.py           ← Qdrant client wrapper
├── rag/
│   ├── retriever.py              ← embed query → search Qdrant → return ranked hits
│   └── formatters.py             ← format retrieved chunks with line numbers
├── nolima/
│   ├── questions.py              ← extract questions from needle_set.json
│   ├── needles.py                ← needle utilities
│   └── char_assigner.py          ← assign characters to needle slots
├── chunking/
│   └── line_chunker.py           ← split haystack text into line-based chunks
├── injection/
│   └── inserter.py               ← insert needle chunks into haystack
├── experiments/
│   ├── config.py                 ← experiment config dataclass
│   └── runner.py                 ← experiment runner
└── utils/
    ├── io.py                     ← JSON load/write helpers
    └── rng.py                    ← seeded RNG utilities
```

**Scripts that require PYTHONPATH:**
- `build_questions.py`
- `run_retrieval_ranking.py`
- `retrieval_with_scores.py`
- `get_embeddings_of_needles.py`
- `run_experiment.py`

**Scripts that do NOT require PYTHONPATH** (self-contained or auto-add to sys.path):
- `run_rag_experiment.py` — no `ragwithtopk` imports
- `run_rag_eval.py` — no `ragwithtopk` imports
- `characters_pdf.py` — adds `NoLiMa_based_RAG/` to sys.path automatically
- `plot_rag_answer_heatmap.py`, `plot_rag_lc_delta_heatmap.py`, `needlewise_best_topk_pdf.py`, `compute_rag_accuracy_from_full_summary.py` — no `ragwithtopk` imports

## Quick-Start (End-to-End)

```bash
export PYTHONPATH="$PWD/NoLiMa_based_RAG:$PYTHONPATH"
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."

# ── 1. Build questions with embeddings (skip if questions/questions.json exists) ─
python3 NoLiMa_based_RAG/scripts/build_questions.py \
    --needle_set datasets/NoLiMa/needlesets/needle_set.json \
    --out        NoLiMa_based_RAG/questions/questions.json

# ── 2. Run retrieval ranking (Qdrant must be running) ─────────────────────────
python3 NoLiMa_based_RAG/scripts/run_retrieval_ranking.py \
    --config         NoLiMa_based_RAG/configs/ck.yaml \
    --questions_json NoLiMa_based_RAG/questions/questions.json \
    --out_dir        NoLiMa_based_RAG/results/NeedleRanking \
    --save_hits

# ── 2b. Generate full binary accuracy table (gitignored — must run locally) ───
python3 NoLiMa_based_RAG/scripts/compute_rag_accuracy_from_full_summary.py \
    --base-csv          NoLiMa_based_RAG/results/results_summary_full.csv \
    --needle-json       NoLiMa_based_RAG/results/NeedleRanking/needle_rank_summary.json \
    --out-csv           NoLiMa_based_RAG/tables/rag_accuracy_full_binary_from_results_summary_full.csv \
    --missing-report-csv NoLiMa_based_RAG/results/missing_rows_results_summary_full_vs_needle.csv

# ── 3. Run RAG experiment (dry run first) ─────────────────────────────────────
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py --dry-run
python3 NoLiMa_based_RAG/scripts/run_rag_experiment.py

# ── 4. Generate plots (uses pre-existing tables) ──────────────────────────────
python3 NoLiMa_based_RAG/scripts/plot_rag_answer_heatmap.py \
    --input-csv NoLiMa_based_RAG/rag_experiment_results/tables/rag_results_raw.csv \
    --out-dir   NoLiMa_based_RAG/plots

python3 NoLiMa_based_RAG/scripts/plot_rag_lc_delta_heatmap.py \
    --lc-csv  NoLiMa_based_RAG/tables/ans_accuracy_selected_contexts.csv \
    --rag-csv NoLiMa_based_RAG/rag_experiment_results/tables/rag_results_raw.csv \
    --out-dir NoLiMa_based_RAG/plots

python3 NoLiMa_based_RAG/scripts/needlewise_best_topk_pdf.py \
    --input-csv NoLiMa_based_RAG/tables/rag_accuracy_full_binary_from_results_summary_full.csv \
    --out-dir   NoLiMa_based_RAG/plots

python3 NoLiMa_based_RAG/scripts/characters_pdf.py \
    --out-dir NoLiMa_based_RAG/plots
```
